**GameTable.tsx 리팩터링:**

1. **인원수별 배치 로직 함수 추가:**
   셀 번호(1~9)와 그리드 영역 매핑:
   - 셀1=row1/col1, 셀2=row1/col2, 셀3=row1/col3
   - 셀4=row2/col1, 셀5=row2/col2(판돈), 셀6=row2/col3
   - 셀7=row3/col1, 셀8=row3/col2, 셀9=row3/col3

   `getOpponentCells(opponentCount: number): number[]` 함수:
   - 5명(6인방): [1,2,3,4,6]
   - 4명(5인방): [1,3,4,6]
   - 3명(4인방): [1,2,3]
   - 2명(3인방): [1,3]
   - 1명(2인방): [2]

   내 플레이어를 players 배열에서 분리:
   - `me = players.find(p => p.id === myPlayerId)`
   - `opponents = players.filter(p => p.id !== myPlayerId)` (원래 순서 유지)

2. **데스크톱 레이아웃 변경:**
   기존 원형 배치(`absolute` + `rotate/translateY`) 제거.
   CSS Grid 3x3 적용:
   ```
   display: grid;
   grid-template-columns: 1fr 1fr 1fr;
   grid-template-rows: 1fr 1fr 1fr;
   ```
   - 각 opponent를 `getOpponentCells()`가 반환한 셀에 배치 (`gridArea: '{row} / {col}'`)
   - 셀5(row2/col2): 판돈 카드 (기존 중앙 팟 표시)
   - 내 플레이어: `gridArea: '3 / 1 / 4 / 4'` (셀7~9 전체 span, 하단 고정)
   - 각 셀의 내용물은 `flex items-center justify-center`로 중앙 정렬
   - 전체 그리드 높이: `h-full` (부모 전체 채움)

3. **모바일 레이아웃 변경:**
   기존 2열 grid 유지하되:
   - 상단: opponents (grid-cols-2)
   - 중앙: 판돈 (full width)
   - 하단: 내 플레이어 (full width)
   공간 활용 개선을 위해 gap-1 유지, padding 최소화.

4. **빅팟 내부글로우 효과:**
   `pot >= 20000` 조건으로 게임테이블 최외곽 컨테이너에 inset box-shadow 적용:
   ```css
   box-shadow: inset 0 0 80px 20px rgba(245, 158, 11, 0.3),
               inset 0 0 200px 60px rgba(234, 88, 12, 0.15);
   ```
   - amber-500 (#f59e0b) + orange-600 (#ea580c) 조합
   - `transition: box-shadow 0.5s ease` 추가 (팟 변동 시 부드러운 전환)
   - 이 inset shadow를 `absolute inset-0 pointer-events-none z-10` div로 오버레이
