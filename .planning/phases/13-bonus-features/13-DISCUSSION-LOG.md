## 영역 1: 기리 스트리밍 범위

**질문:** 기리 플레이어의 어떤 액션을 실시간으로 브로드캐스트할까요?

| 옵션 | 설명 |
|------|------|
| 더미 상태 전체 동기화 | 드래그/탭 액션마다 piles[] + tapOrder emit |
| **단계 전환만 동기화** ✅ | split→tap→merging→done 4단계 전환 시점만 emit |
| 최종 결과만 (현행 유지) | 기존 cut 이벤트만 유지, 중간 상태 없음 |

**선택:** 단계 전환만 동기화
**사용자 노트:** 데스크탑과 모바일의 기리 UI가 다른 것도 주의해야 하는 포인트. 근본적으로 다르게 구현됨.

## 영역 3: SFX 이벤트 매핑

**질문:** 게임 이벤트와 SFX 파일 매핑 방식

| 옵션 | 설명 |
|------|------|
| 코드 하드코딩 | 개발자가 직접 코드에 매핑 작성 |
| **sfx-mapping.md 템플릿** ✅ | 플래너가 템플릿 생성, 사용자가 파일명 채워넣기 |

**선택:** sfx-mapping.md 템플릿
**사용자 노트:** sfx는 정리 전혀 안 되고 모아놓기만 한 것. 매핑 md 문서 만들어놓으면 내가 파일명 적는 방식으로 진행.

## 영역 4: BGM 제어 UI

**질문 1:** BGM 볼륨/음소거 버튼 위치

| 옵션 | 설명 |
|------|------|
| **우상단 고정** ✅ | 항상 보이는 위치 |

| 자동재생, 화면 버튼 없음 | 브라우저 기본 탭 음소거 사용 |

**선택:** 우상단 고정

| 옵션 | 설명 |
|------|------|
| BGM만 버튼, SFX 항상 ON | SFX는 별도 제어 없음 |
| **각각 별도 버튼** ✅ | BGM 음악 아이콘 + SFX 스피커 아이콘 |

**선택:** 각각 별도 버튼 (음악 아이콘 + 스피커 아이콘)
