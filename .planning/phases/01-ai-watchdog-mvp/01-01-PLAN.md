**3. /opt/ai-watchdog/watchdog.py** 구현 사양:

(a) 로그 수집 함수 `collect_logs()`:
- `subprocess.run(["journalctl", "-u", "<실제PM2유닛명>", "--since", "1 hour ago", "--no-pager", "-n", str(WATCHDOG_LOG_LINES)])` 로 앱 로그 수집
- `subprocess.run(["tail", "-n", str(WATCHDOG_LOG_LINES), "/var/log/nginx/error.log"])` 로 nginx 에러 로그 수집
- 각 소스를 딕셔너리 `{"journalctl": str, "nginx_error": str}`로 반환
- subprocess 실패 시 해당 소스는 빈 문자열로 처리 (try/except)

(b) 필터 함수 `filter_logs(raw_logs: dict) -> list[dict]`:
- CRITICAL_PATTERNS (브리핑 초안 기반 + 실제 VM 확인 후 조정):
  ```python
  CRITICAL_PATTERNS = [
      r"UnhandledPromiseRejection",
      r"ECONNREFUSED",
      r"EADDRINUSE",
      r"heap out of memory",
      r"SIGTERM|SIGKILL",
      r"upstream timed out",
      r"connect\(\) failed",
      r"no live upstreams",
      r"ERR_PNPM",
      r"Error:|error:|ERROR",
      r"FATAL",
      r"OOM|OutOfMemory",
  ]
  ```
- NOISE_PATTERNS:
  ```python
  NOISE_PATTERNS = [
      r"GET /health",
      r"GET /favicon",
      r"socket\.io.*polling",
      r"::ffff:",
      r"statusCode.*200",
  ]
  ```
- 각 로그 라인에 대해: NOISE 매칭 시 스킵 -> CRITICAL 매칭 시 `{"source": str, "line": str, "pattern": str}` 리스트에 추가
- 매칭 결과가 0건이면 빈 리스트 반환

(c) 시스템 컨텍스트 수집 `collect_system_info() -> str`:
- `free -h`, `df -h /`, `uptime`, `pm2 jlist` (JSON에서 메모리/CPU 추출)
- 각 명령 실패 시 "N/A"로 대체
- 한 줄 요약 문자열 반환

(d) Haiku 분석 함수 `analyze_with_haiku(filtered_logs: list, system_info: str) -> str`:
- `from anthropic import Anthropic` 사용
- 모델: `claude-haiku-4-5`
- max_tokens: 1024
- system prompt: "너는 서버 로그 분석 전문가다. 아래 로그를 분석하고 severity(1-5), 원인 추정, 조치 제안을 JSON 배열로 응답해. 각 항목: {severity: int, summary: str, cause: str, action: str}"
- user message: filtered_logs (최대 WATCHDOG_CONTEXT_LINES줄) + system_info
- API 호출 실패 시 에러 메시지를 문자열로 반환 (예외 발생하지 않음)
- 분당 3회 제한: 단일 실행이므로 1회 호출만 수행 (rate limit 문제 없음)

(e) MD 출력 함수 `write_report(analysis: str, filtered_logs: list, system_info: str)`:
- 출력 경로: `/opt/ai-watchdog/.debug/YYYY-MM-DD-HH.md` (per D-03)
- `.debug/` 디렉토리 없으면 `os.makedirs` 생성
- 파일명: `datetime.now().strftime("%Y-%m-%d-%H")` + ".md"
- MD 포맷:
  ```markdown
  # Watchdog Report {YYYY-MM-DD HH:00}

  ## System Info
  {system_info}

  ## Filtered Logs ({N} entries)
  {각 로그 항목: source, pattern, line}

  ## AI Analysis
  {haiku 응답 원문}

  ---
  Generated by ai-watchdog at {ISO timestamp}
  ```
- severity_threshold 필터: Haiku 응답을 JSON 파싱하여 severity >= WATCHDOG_SEVERITY_THRESHOLD 항목만 포함. 파싱 실패 시 원문 그대로 출력.

(f) main 함수:
- dotenv.load_dotenv() 로 .env 로드
- ANTHROPIC_API_KEY 없으면 에러 메시지 출력 후 exit(1)
- collect_logs -> filter_logs -> (매칭 0건이면 "No critical logs found" 출력 후 정상 종료) -> collect_system_info -> analyze_with_haiku -> write_report
- 최종 출력: 생성된 MD 파일 경로 print

VM에 파일 배포 방법: SSH heredoc 또는 scp로 전송. pip install은 Task 2에서 처리.

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
LOG_FILE="$SCRIPT_DIR/watchdog.log"

echo "[$(date -Iseconds)] Watchdog started" >> "$LOG_FILE"

cd "$SCRIPT_DIR"
/usr/bin/python3 "$SCRIPT_DIR/watchdog.py" >> "$LOG_FILE" 2>&1

EXIT_CODE=$?
echo "[$(date -Iseconds)] Watchdog finished (exit=$EXIT_CODE)" >> "$LOG_FILE"
exit $EXIT_CODE
```
- chmod +x run-watchdog.sh
