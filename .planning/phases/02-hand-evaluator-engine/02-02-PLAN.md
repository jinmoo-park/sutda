**목적:** 두 `HandResult`를 비교해 승자를 판정한다. 동점이면 `'tie'`를 반환한다.

**함수 시그니처:**
```typescript
import type { HandResult } from '../types/hand';

export function compareHands(a: HandResult, b: HandResult): 'a' | 'b' | 'tie'
```

---

**compareHands 비교 로직 (우선순위 순서로 처리):**

#### 케이스 1: 둘 다 일반 패 (isSpecialBeater=false)
단순 score 비교:
```
a.score > b.score → 'a'
a.score < b.score → 'b'
a.score === b.score → 'tie'
```

#### 케이스 2: 한쪽이 땡잡이 (isSpecialBeater=true, score=0, rank=[3,7])
땡잡이 판별 헬퍼 (compare.ts 내부):
```typescript
function isTtaengJabi(h: HandResult): boolean {
  // evaluateHand가 땡잡이를 score=0, isSpecialBeater=true로 반환
  // 암행어사와 구별: 암행어사는 score=1
  return h.isSpecialBeater && h.score === 0;
}
```

땡잡이 규칙:
```
상대방이 장땡/광땡 (score >= 1010) → 땡잡이가 짐 ('b' if a is 땡잡이, 'a' if b is 땡잡이)
상대방이 구땡~일땡 (1001 <= score <= 1009) → 땡잡이가 이김
상대방이 특수조합/끗 (score < 1001) 또는 상대도 땡잡이 → score로 비교 (둘 다 0이면 'tie')
```

**중요:** 땡잡이 vs 땡잡이는 동점 처리 → `'tie'`

#### 케이스 3: 한쪽이 암행어사 (isSpecialBeater=true, score=1, rank=[4열끗,7열끗])
암행어사 판별 헬퍼:
```typescript
function isAmhaengEosa(h: HandResult): boolean {
  return h.isSpecialBeater && h.score === 1;
}
```

암행어사 규칙:
```
상대방이 삼팔광땡 (score === 1300) → 암행어사가 짐
상대방이 일팔광땡 (score === 1200) 또는 일삼광땡 (score === 1100) → 암행어사가 이김
상대방이 그 외 모든 패 → score로 비교 (1끗으로 인식)
```

**중요:** 암행어사 vs 암행어사는 동점 처리 → `'tie'`
**주의:** 암행어사 vs 땡잡이: 둘 다 isSpecialBeater=true → 각각의 특수 규칙 적용
  - 암행어사 score=1, 땡잡이 score=0 → 어떤 땡에도 해당 안 함 → score 비교 → 암행어사(1) > 땡잡이(0) → 암행어사 승

#### 케이스 4: 동점 (HAND-07)
score가 같고 특수 규칙도 동점인 경우 → `'tie'` 반환
(D-01: 재경기 트리거, 실제 재경기 처리는 게임 로직에서 담당, compareHands는 단순히 'tie' 반환)

---

**compareHands 내부 처리 순서 (의사코드):**
```
function compareHands(a, b):
  aIsTA = isTtaengJabi(a)
  bIsTA = isTtaengJabi(b)
  aIsAE = isAmhaengEosa(a)
  bIsAE = isAmhaengEosa(b)

  // 땡잡이 특수 처리
  if aIsTA and not bIsTA:
    if b.score >= 1010: return 'b'    // 장땡/광땡에게 짐
    if b.score >= 1001: return 'a'    // 구땡~일땡에게 이김
    return scoreCompare(a, b)          // 특수조합/끗 → 망통(0)으로 비교

  if bIsTA and not aIsTA:
    if a.score >= 1010: return 'a'
    if a.score >= 1001: return 'b'
    return scoreCompare(a, b)

  // 암행어사 특수 처리
  if aIsAE and not bIsAE:
    if b.score === 1300: return 'b'   // 삼팔광땡에게 짐
    if b.score === 1200 or b.score === 1100: return 'a'  // 일팔/일삼광땡에게 이김
    return scoreCompare(a, b)          // 나머지 → 1끗으로 비교

  if bIsAE and not aIsAE:
    if a.score === 1300: return 'a'
    if a.score === 1200 or a.score === 1100: return 'b'
    return scoreCompare(a, b)

  // 둘 다 특수패이거나 둘 다 일반패 → score 비교
  return scoreCompare(a, b)

function scoreCompare(a, b):
  if a.score > b.score: return 'a'
  if a.score < b.score: return 'b'
  return 'tie'
```

**checkGusaTrigger 함수 시그니처:**
```typescript
import type { HandResult } from '../types/hand';

export function checkGusaTrigger(
  gusaHand: HandResult,
  allSurvivingHands: HandResult[]
): { shouldRedeal: boolean }
```

**재경기 트리거 조건 (D-07 기반):**

구사 재경기는 해당 구사 패가 **생존자 중 최고 패보다 강하거나 같을 때** 트리거된다.
`allSurvivingHands`는 구사 패를 포함한 모든 생존자 패 배열이다.

```
일반 구사 (isGusa=true):
  재경기 조건: allSurvivingHands 중 최고 score가 알리(score=60) 이하
  즉, 어떤 생존자도 독사(50) 이상 특수조합을 갖지 않아야 함
  → 최고 score <= 60 이면 shouldRedeal: true

멍텅구리 구사 (isMeongtteongguriGusa=true):
  재경기 조건: allSurvivingHands 중 최고 score가 팔땡(score=1008) 이하
  즉, 구땡(1009) 이상 강한 패가 없어야 함
  → 최고 score <= 1008 이면 shouldRedeal: true
```

**예외 처리:**
- `gusaHand`가 구사도 멍텅구리구사도 아닌 경우 (`isGusa=false && isMeongtteongguriGusa=false`): `shouldRedeal: false`
- `allSurvivingHands`가 비어있는 경우: `shouldRedeal: false`

**의사코드:**
```
function checkGusaTrigger(gusaHand, allSurvivingHands):
  if not gusaHand.isGusa and not gusaHand.isMeongtteongguriGusa:
    return { shouldRedeal: false }
  if allSurvivingHands.length === 0:
    return { shouldRedeal: false }

  maxScore = max(allSurvivingHands.map(h => h.score))

  if gusaHand.isGusa:
    return { shouldRedeal: maxScore <= 60 }  // 알리(60) 이하

  if gusaHand.isMeongtteongguriGusa:
    return { shouldRedeal: maxScore <= 1008 }  // 팔땡(1008) 이하

  return { shouldRedeal: false }
```

## 구현 노트

### 땡잡이 vs 암행어사 구별 방법
`evaluateHand`에서:
- 땡잡이 → `{ score: 0, isSpecialBeater: true }`
- 암행어사 → `{ score: 1, isSpecialBeater: true }`

`compareHands`에서 두 특수패를 구별하는 가장 간단한 방법은 score 값이다:
- `isSpecialBeater && score === 0` → 땡잡이
- `isSpecialBeater && score === 1` → 암행어사

더 명시적인 구별을 원한다면 `HandResult`에 `specialBeaterType?: 'ttaeng-jabi' | 'amhaeng-eosa'` 필드를 추가할 수 있지만,
D-04에서 정의된 인터페이스에 없으므로 score 구별 방식을 사용한다.

### 암행어사 vs 장땡 케이스 주의
암행어사는 **광땡만** 이기는 특수 규칙이 적용된다:
- 일팔광땡(1200) → 암행어사 승
- 일삼광땡(1100) → 암행어사 승
- 삼팔광땡(1300) → 암행어사 짐
- 장땡(1010) → score 비교 → 1 < 1010 → 암행어사 짐
- 구땡~일땡 → score 비교 → 암행어사 짐

즉, 암행어사 특수 우선 규칙은 score 1100~1300 범위(광땡)에만 적용된다.

### 구사 재경기 트리거 기준점 확인
- 일반 구사 기준: **알리(60) 이하** → score <= 60
  - 알리(60) ← 포함 O
  - 독사(50), 구삥(40), 장삥(30), 장사(20), 새륙(10), 끗(0~9) ← 포함 O
  - 일땡(1001) 이상 ← 포함 X
- 멍텅구리구사 기준: **팔땡(1008) 이하** → score <= 1008
  - 팔땡(1008) ← 포함 O
  - 칠땡(1007), ..., 일땡(1001), 알리~끗 ← 포함 O
  - 구땡(1009) 이상 ← 포함 X

### `allSurvivingHands`는 구사 패 자신 포함
`checkGusaTrigger`를 호출할 때 `allSurvivingHands`는 구사 패 자신도 포함한다.
구사 패의 score(3끗 = score 3)는 알리(60)보다 작으므로, 자신만 있는 경우에도 재경기 트리거가 된다.

### 파일 구조 (완성 후)
```
packages/shared/src/
├── types/
│   ├── card.ts           ← 기존 (수정 없음)
│   ├── game.ts           ← 기존 (수정 없음)
│   └── hand.ts           ← Plan 1 생성
├── hand/
│   ├── evaluator.ts      ← Plan 1 생성
│   ├── evaluator.test.ts ← Plan 1 생성
│   ├── compare.ts        ← 이 플랜 생성 (Task 1)
│   ├── compare.test.ts   ← 이 플랜 생성 (Task 1)
│   ├── gusa.ts           ← 이 플랜 생성 (Task 2)
│   └── gusa.test.ts      ← 이 플랜 생성 (Task 2)
├── deck.ts               ← 기존 (수정 없음)
├── deck.test.ts          ← 기존 (수정 없음)
└── index.ts              ← 이 플랜에서 exports 추가 (Task 2)
```
