## 작업 목록

### Task 1: HandType enum + HandResult interface 타입 정의

**파일:** `packages/shared/src/types/hand.ts`
**목적:** 족보 판정 시스템 전체가 공유하는 타입 계약(contract)을 먼저 정의한다

**구체적 구현 명세:**

```typescript
/** 섯다 족보 종류 - 숫자가 높을수록 강한 패 */
export type HandType =
  // 광땡 (가장 강함)
  | 'sam-pal-gwang-ttaeng'    // 삼팔광땡: 3광 + 8광
  | 'il-pal-gwang-ttaeng'     // 일팔광땡: 1광 + 8광
  | 'il-sam-gwang-ttaeng'     // 일삼광땡: 1광 + 3광
  // 장땡~일땡
  | 'jang-ttaeng'             // 장땡: 10 + 10
  | 'gu-ttaeng'               // 구땡: 9 + 9
  | 'pal-ttaeng'              // 팔땡: 8 + 8
  | 'chil-ttaeng'             // 칠땡: 7 + 7
  | 'yuk-ttaeng'              // 육땡: 6 + 6
  | 'o-ttaeng'                // 오땡: 5 + 5
  | 'sa-ttaeng'               // 사땡: 4 + 4
  | 'sam-ttaeng'              // 삼땡: 3 + 3
  | 'i-ttaeng'                // 이땡: 2 + 2
  | 'il-ttaeng'               // 일땡: 1 + 1
  // 특수 조합 (알리~새륙)
  | 'ali'                     // 알리: 1 + 2
  | 'dok-sa'                  // 독사: 1 + 4
  | 'gu-bbing'                // 구삥: 1 + 9
  | 'jang-bbing'              // 장삥: 1 + 10
  | 'jang-sa'                 // 장사: 10 + 4
  | 'sae-ryuk'                // 새륙: 4 + 6
  // 끗 (0~9)
  | 'kkut';                   // 망통(0끗)~아홉끗(9끗)

/** 족보 판정 결과 */
export interface HandResult {
  /** 족보 종류 */
  handType: HandType;
  /**
   * 비교용 점수 (높을수록 강함).
   * 광땡: 1000+, 땡: 100+rank, 특수조합: 고정값, 끗: 0~9
   *
   * 점수 체계:
   * - 삼팔광땡: 1300
   * - 일팔광땡: 1200
   * - 일삼광땡: 1100
   * - 장땡: 1010, 구땡: 1009, ..., 일땡: 1001
   * - 알리: 60, 독사: 50, 구삥: 40, 장삥: 30, 장사: 20, 새륙: 10
   * - 끗: 0~9 (일의 자리 합)
   */
  score: number;
  /** 땡잡이 또는 암행어사 여부 - true면 특수 비교 로직 적용 */
  isSpecialBeater: boolean;
  /** 일반 구사 여부 (4일반 + 9일반) */
  isGusa: boolean;
  /** 멍텅구리구사 여부 (4열끗 + 9열끗) */
  isMeongtteongguriGusa: boolean;
}
```

**점수 체계 설계 근거:**
- 광땡은 1100~1300으로 땡(1001~1010)보다 큰 값 사용
- 특수 조합은 60~10으로 끗(0~9)보다 큰 값 사용
- 끗은 그대로 0~9 사용 (망통=0)
- 땡잡이/암행어사는 `isSpecialBeater=true` + score는 맥락에 따른 fallback 값 사용
  - 땡잡이: score=0 (끗/특수 상대 시 망통 취급), handType='kkut'
  - 암행어사: score=1 (나머지 상대 시 1끗 취급), handType='kkut'
  - 단, `isSpecialBeater=true`이면 compareHands에서 특수 비교 로직이 우선 적용됨

**`evaluateHand` 함수 시그니처:**
```typescript
import type { Card } from '../types/card';
import type { HandResult } from '../types/hand';

export function evaluateHand(card1: Card, card2: Card): HandResult
```

**판정 우선순위 로직 (순서대로 체크):**

```
1. 두 카드 모두 gwang 속성인가?
   - ranks가 [3,8] → 삼팔광땡 (score: 1300)
   - ranks가 [1,8] → 일팔광땡 (score: 1200)
   - ranks가 [1,3] → 일삼광땡 (score: 1100)

2. 두 카드의 rank가 같은가?
   - rank가 10 → 장땡 (score: 1010)
   - rank가 9 → 구땡 (score: 1009)
   - ...
   - rank가 1 → 일땡 (score: 1001)

3. 특수패 체크 (rank 조합 기준):
   - 암행어사: ranks가 [4,7] AND 두 카드 모두 yeolkkeut 속성
     → isSpecialBeater=true, score=1, handType='kkut'
   - 땡잡이: ranks가 [3,7] AND 두 카드 모두 normal 속성
     → isSpecialBeater=true, score=0, handType='kkut'
   - 멍텅구리구사: ranks가 [4,9] AND 두 카드 모두 yeolkkeut 속성
     → isMeongtteongguriGusa=true, score=끗계산(3), handType='kkut'
   - 일반구사: ranks가 [4,9] AND 두 카드 모두 normal 속성
     → isGusa=true, score=끗계산(3), handType='kkut'

4. 특수 조합 (rank 조합 기준, 속성 무관):
   - [1,2] → 알리 (score: 60)
   - [1,4] → 독사 (score: 50)
   - [1,9] → 구삥 (score: 40)
   - [1,10] → 장삥 (score: 30)
   - [4,10] → 장사 (score: 20)
   - [4,6] → 새륙 (score: 10)

5. 끗: (rank1 + rank2) % 10
   - handType='kkut', score=끗 값 (0~9)
```

**주요 엣지 케이스 처리:**
- 광땡 판정은 반드시 두 카드 모두 `attribute === 'gwang'`인지 확인
- 땡잡이 판정: [3,7] ranks이지만 3광+7normal이면 땡잡이가 아님 → 끗(0끗)으로 처리
- 암행어사 판정: [4,7] ranks이지만 4열끗+7normal이면 암행어사 아님 → 새륙/끗으로 처리
- 특수 조합(알리~새륙)과 끗에서는 `isSpecialBeater=false`, `isGusa=false`, `isMeongtteongguriGusa=false`
- rank 10인 카드는 항상 normal 속성 (YEOLKKEUT_RANKS에 없음) → [4,10]은 장사

## 테스트 케이스

`packages/shared/src/hand/evaluator.test.ts`에 포함해야 할 테스트:

### 광땡 테스트
```typescript
// 삼팔광땡
evaluateHand({ rank: 3, attribute: 'gwang' }, { rank: 8, attribute: 'gwang' })
// → { handType: 'sam-pal-gwang-ttaeng', score: 1300, isSpecialBeater: false, isGusa: false, isMeongtteongguriGusa: false }

// 일팔광땡 (카드 순서 반대)
evaluateHand({ rank: 8, attribute: 'gwang' }, { rank: 1, attribute: 'gwang' })
// → { handType: 'il-pal-gwang-ttaeng', score: 1200, ... }

// 일삼광땡
evaluateHand({ rank: 1, attribute: 'gwang' }, { rank: 3, attribute: 'gwang' })
// → { handType: 'il-sam-gwang-ttaeng', score: 1100, ... }
```

### 땡 테스트
```typescript
// 장땡 (10 normal + 10 normal)
evaluateHand({ rank: 10, attribute: 'normal' }, { rank: 10, attribute: 'normal' })
// → { handType: 'jang-ttaeng', score: 1010, ... }

// 구땡 (9열끗 + 9normal)  ← 같은 rank이면 땡 (속성 무관)
evaluateHand({ rank: 9, attribute: 'yeolkkeut' }, { rank: 9, attribute: 'normal' })
// → { handType: 'gu-ttaeng', score: 1009, ... }

// 일땡
evaluateHand({ rank: 1, attribute: 'gwang' }, { rank: 1, attribute: 'normal' })
// → { handType: 'il-ttaeng', score: 1001, ... }
```

### 특수 조합 테스트
```typescript
// 알리
evaluateHand({ rank: 1, attribute: 'gwang' }, { rank: 2, attribute: 'normal' })
// → { handType: 'ali', score: 60, ... }

// 독사 (1normal + 4normal) ← 속성 달라도 독사
evaluateHand({ rank: 1, attribute: 'normal' }, { rank: 4, attribute: 'normal' })
// → { handType: 'dok-sa', score: 50, ... }

// 독사 (1광 + 4열끗) ← 속성 달라도 rank [1,4]면 독사
evaluateHand({ rank: 1, attribute: 'gwang' }, { rank: 4, attribute: 'yeolkkeut' })
// → { handType: 'dok-sa', score: 50, ... }

// 구삥
evaluateHand({ rank: 9, attribute: 'normal' }, { rank: 1, attribute: 'gwang' })
// → { handType: 'gu-bbing', score: 40, ... }

// 장삥
evaluateHand({ rank: 10, attribute: 'normal' }, { rank: 1, attribute: 'gwang' })
// → { handType: 'jang-bbing', score: 30, ... }

// 장사
evaluateHand({ rank: 10, attribute: 'normal' }, { rank: 4, attribute: 'normal' })
// → { handType: 'jang-sa', score: 20, ... }

// 새륙
evaluateHand({ rank: 4, attribute: 'normal' }, { rank: 6, attribute: 'normal' })
// → { handType: 'sae-ryuk', score: 10, ... }
```

### 특수패 테스트 (땡잡이, 암행어사, 구사)
```typescript
// 땡잡이: 3normal + 7normal
evaluateHand({ rank: 3, attribute: 'normal' }, { rank: 7, attribute: 'normal' })
// → { handType: 'kkut', score: 0, isSpecialBeater: true, isGusa: false, isMeongtteongguriGusa: false }

// 땡잡이 아님: 3광 + 7normal → 끗 (3+7=10 → 0끗)
evaluateHand({ rank: 3, attribute: 'gwang' }, { rank: 7, attribute: 'normal' })
// → { handType: 'kkut', score: 0, isSpecialBeater: false, isGusa: false, isMeongtteongguriGusa: false }

// 암행어사: 4열끗 + 7열끗
evaluateHand({ rank: 4, attribute: 'yeolkkeut' }, { rank: 7, attribute: 'yeolkkeut' })
// → { handType: 'kkut', score: 1, isSpecialBeater: true, isGusa: false, isMeongtteongguriGusa: false }

// 암행어사 아님: 4열끗 + 7normal → 끗 (4+7=11 → 1끗)
evaluateHand({ rank: 4, attribute: 'yeolkkeut' }, { rank: 7, attribute: 'normal' })
// → { handType: 'kkut', score: 1, isSpecialBeater: false, isGusa: false, isMeongtteongguriGusa: false }

// 일반 구사: 4normal + 9normal
evaluateHand({ rank: 4, attribute: 'normal' }, { rank: 9, attribute: 'normal' })
// → { handType: 'kkut', score: 3, isSpecialBeater: false, isGusa: true, isMeongtteongguriGusa: false }

// 멍텅구리 구사: 4열끗 + 9열끗
evaluateHand({ rank: 4, attribute: 'yeolkkeut' }, { rank: 9, attribute: 'yeolkkeut' })
// → { handType: 'kkut', score: 3, isSpecialBeater: false, isGusa: false, isMeongtteongguriGusa: true }

// 구사 아님: 4열끗 + 9normal (속성 혼합) → 끗 3끗
evaluateHand({ rank: 4, attribute: 'yeolkkeut' }, { rank: 9, attribute: 'normal' })
// → { handType: 'kkut', score: 3, isSpecialBeater: false, isGusa: false, isMeongtteongguriGusa: false }
```

### 끗 테스트

// 망통 (0끗): 합이 10의 배수

## 구현 노트

### 카드 정렬 헬퍼 사용 권장
`evaluateHand` 내부에서 두 카드의 rank를 항상 정렬(`[min, max]`)해서 비교하면 조건문이 단순해진다:
```typescript
const ranks = [card1.rank, card2.rank].sort((a, b) => a - b) as [CardRank, CardRank];
const [low, high] = ranks;
```

### 특수패 판정 순서 주의사항
- **광땡 체크 → 땡 체크 → 특수패(암행어사/땡잡이/구사) → 특수조합 → 끗** 순서를 반드시 지켜야 한다
- 예: [4,7] ranks이지만 암행어사 체크보다 땡 체크가 먼저 오면 문제 없음 (rank가 같아야 땡이므로)
- 예: [4,9] ranks의 경우 구사/멍텅구리구사 체크를 특수조합 앞에 놓아야 함 (독사를 [1,4]로 판정하므로 [4,9]는 해당 없지만, 구사를 먼저 잡아야 끗으로 빠지지 않음)

### 독사/장사 엣지케이스: 속성 무관
- `[1,4]` ranks인 경우 4가 `yeolkkeut`이어도 독사 (암행어사와 달리 속성 무관)
- `[4,10]` ranks인 경우 4가 `yeolkkeut`이어도 장사
- 단, `[4,9]` ranks에서 구사/멍텅구리구사는 속성이 중요하므로 주의

### `isSpecialBeater` 카드의 score 의미
- 땡잡이(score=0, handType='kkut'): 끗/특수조합 상대 시 망통 취급 → compareHands에서 score로 비교 시 지는 것이 올바른 동작
- 암행어사(score=1, handType='kkut'): 나머지 상대 시 1끗 취급 → compareHands에서 score로 비교 시 1끗 동작
- `compareHands`에서 `isSpecialBeater=true`인 경우 특수 로직을 먼저 확인해야 함 (Plan 2 담당)
