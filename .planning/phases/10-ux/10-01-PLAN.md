    **cardImageUtils.ts 생성:**

    Card의 rank와 attribute를 이미지 파일 경로로 매핑하는 순수 함수들.

    파일명 규칙: `{rank 2자리 패딩}-{fileIndex}.png`
    - fileIndex 결정 규칙: 광(gwang) 또는 열끗(yeolkkeut) 속성의 카드 = `-1`, 일반(normal) = `-2`
    - 단, rank별 유일한 속성이 있는 경우 (rank 2,5,6,10은 항상 normal 2장):
      - rank 2: 02-1.png, 02-2.png (둘 다 normal) → 첫 번째 카드가 -1, 두 번째가 -2
    - 실제 파일 매핑 (img/ 폴더 파일 기준):
      - rank 1: gwang→01-1, normal→01-2
      - rank 2: normal→02-1, normal→02-2 (같은 속성 2장 → fileIndex 계산 필요)
      - rank 3: gwang→03-1, normal→03-2
      - rank 4: yeolkkeut→04-1, normal→04-2
      - rank 5: normal→05-1, normal→05-2
      - rank 6: normal→06-1, normal→06-2
      - rank 7: yeolkkeut→07-1, normal→07-2
      - rank 8: gwang→08-1, normal→08-2
      - rank 9: yeolkkeut→09-1, normal→09-2
      - rank 10: normal→10-1, normal→10-2

    결론: 특수 속성(gwang, yeolkkeut)이 있는 rank는 특수=-1, normal=-2.
    양쪽 모두 normal인 rank(2,5,6,10)는 구별이 불가하므로 — 서버에서 카드를 createDeck으로 생성할 때 같은 rank의 두 카드가 순서대로 나오는데, 이미지 구별이 불필요 (둘 다 같은 숫자 화투). **해결: normal 2장인 rank는 첫 번째 카드를 -1, 두 번째를 -2로 매핑하면 되지만, Card 인터페이스에 순서 정보가 없으므로 둘 다 -1로 매핑하거나 추가 인자를 받는다.**

    최종 구현: `getCardImageSrc(card: Card, slotIndex?: number): string`
    - 특수 속성 카드(gwang/yeolkkeut): 항상 `-1`
    - normal 속성 카드:
      - 해당 rank에 특수 속성 카드가 존재하면(rank 1,3,4,7,8,9): `-2`
      - 해당 rank가 모두 normal이면(rank 2,5,6,10): slotIndex로 구분 (0→`-1`, 1→`-2`, 기본값 `-1`)

    ```typescript
    const SPECIAL_RANKS = new Set([1, 3, 4, 7, 8, 9]); // 이 rank는 특수+normal 조합

    export function getCardImageSrc(card: Card, slotIndex: number = 0): string {
      const paddedRank = String(card.rank).padStart(2, '0');
      let fileIndex: number;
      if (card.attribute !== 'normal') {
        fileIndex = 1; // gwang, yeolkkeut → 항상 -1
      } else if (SPECIAL_RANKS.has(card.rank)) {
        fileIndex = 2; // 이 rank의 normal 카드는 항상 -2
      } else {
        fileIndex = slotIndex === 1 ? 2 : 1; // 모두 normal인 rank → slotIndex로 구분
      }
      return `/img/${paddedRank}-${fileIndex}.png`;
    }

    export function getCardBackSrc(): string {
      return '/img/card_back.jpg';
    }
    ```

    Vite의 기본 publicDir은 packages/client/public/ 인데, 이미지는 루트 img/에 있다.

    **해결책: Vite publicDir을 루트로 설정하지 않고, img/ 폴더를 packages/client/public/img/ 로 심볼릭 링크 생성.**

    하지만 Windows에서 심링크는 관리자 권한 이슈가 있을 수 있다. 더 안전한 방법:

    **Vite 설정에서 publicDir을 프로젝트 루트로 지정:**
    ```typescript
    import path from "path";
    import tailwindcss from "@tailwindcss/vite";
    import react from "@vitejs/plugin-react";
    import { defineConfig } from "vite";

    export default defineConfig({
      plugins: [react(), tailwindcss()],
      publicDir: path.resolve(__dirname, '../../'),  // 프로젝트 루트를 public으로
      resolve: {
        alias: {
          "@": path.resolve(__dirname, "./src"),
        },
      },
      server: {
        allowedHosts: true,
        proxy: {
          '/socket.io': {
            target: 'http://localhost:3001',
            ws: true,
            changeOrigin: true,
          },
        },
      },
    });
    ```

    **문제:** 프로젝트 루트 전체를 publicDir로 하면 불필요한 파일(node_modules 등)이 빌드에 포함될 수 있다.

    **더 좋은 해결책:** packages/client/public/img 디렉토리에 실제 이미지를 복사하지 않고, Vite의 `server.fs.allow`와 커스텀 플러그인 대신 **단순히 packages/client/public/img/ 디렉토리를 생성하고 루트 img/ 파일들을 복사한다.** 빌드 시에만 필요하고, dev에서는 proxy로도 가능하지만 가장 단순한 방법으로 간다.

    **최종 방법 (가장 단순):** `packages/client/public/img/` 폴더 생성 후 루트 `img/` 의 이미지 파일을 복사. vite.config.ts는 변경 불필요 (기본 publicDir = public/).

    실행 순서:
    1. `mkdir -p packages/client/public/img`
    2. `cp img/*.png img/*.jpg packages/client/public/img/`
    3. `.gitignore`에 `packages/client/public/img/` 추가 (빌드 아티팩트)
    4. `package.json`에 prebuild 스크립트 추가 또는 README에 기록

    **하지만 이미지를 git에서 추적해야 하므로 gitignore는 부적절.** 대신 이미지가 이미 루트 img/에 있으므로:

    **최종 결정: vite.config.ts에서 publicDir만 변경:**
    ```typescript
    publicDir: path.resolve(__dirname, '../../img'),
    ```
    이렇게 하면 `img/` 폴더 내용이 dev 서버 루트에서 서빙된다. 즉 `/01-1.png`로 접근 가능.

    **최종 최종 결정:** vite.config.ts 변경 없이, 루트 img/ 폴더를 packages/client/public/ 아래로 이동하거나 심링크. **가장 깔끔한 방법은 Vite 설정 미변경 + packages/client/public/img 심링크 생성:**

    ```bash
    # Windows에서는 junction 사용 (관리자 권한 불필요)
    mklink /J packages\client\public\img ..\..\img
    ```

    **진짜 최종 결정 (Vite 공식 방법):**

    ```typescript
    export default defineConfig({
    ```

    **vite.config.ts publicDir을 루트로 설정. 빌드 시 루트의 모든 파일이 아닌 특정 패턴만 복사되도록 build.copyPublicDir 활용. 하지만 Vite는 publicDir 전체를 복사한다.**
