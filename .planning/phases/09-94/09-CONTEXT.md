<decisions>
## Implementation Decisions

### 땡값 (MODE-OG-03, RULE-04)

- **D-01 적용 모드:** 오리지날 모드(`'original'`)에서만 땡값 적용. 세장/한장공유/골라골라/인디언은 땡값 없음.
  94재경기는 모든 모드에서 트리거 가능 (아래 RULE-01~03 참조).

- **D-02 트리거 조건:** showdown 후 result phase 진입 시 자동 계산. **최종 승자가 땡(일땡 이상)으로 승리했을 때**, 해당 판에 다이한 **모든** 플레이어가 승자에게 땡값을 납부한다. 납부 금액은 승자의 땡 등급으로 결정: 일땡~구땡 승리 → 500원, 장땡/삼팔광땡/일팔광땡/일삼광땡 승리 → 1000원. 다이한 플레이어 자신의 패는 관계없음.

- **D-03 땡값 면제 조건 (RULE-04):** 최종 승자가 **땡잡이(3+7)** 또는 **암행어사(열끗4+열끗7)**로 승리한 경우 땡값 전액 면제. `compareHands` 결과의 `isSpecialBeater` 플래그로 판별 가능.

- **D-04 최후 생존 패배자 면제:** 다이하지 않고 최후까지 베팅에 따라가서 패배한 플레이어는 땡값 납부 대상이 아님. 즉, **다이한 플레이어만** 땡값 납부.

- **D-05 UI 표시:** result phase에서 기존 pot 정산 결과에 **합산한 최종 칩 변동**을 보여주되, 별도로 "땡값 납부: X원" 항목을 result 화면에 표기한다.
  - 예: "김철수: -500원 (땡값)" / "이영희: +500원 (땡값)" 형태로 표시
  - 기존 pot 정산 표시와 구분된 섹션으로 제공

### 94재경기 (RULE-01, RULE-02, RULE-03)

- **D-06 트리거 조건 (모든 게임 모드에서 공통 적용):**
  - **일반 구사 재경기 (RULE-01):** 구사(열끗 아닌 4+9) 보유자가 생존(다이 안 함) 상태이고, 생존자 중 최고패가 **알리 이하**일 때 자동 트리거
  - **멍텅구리구사 재경기 (RULE-02):** 멍텅구리구사(열끗4+열끗9) 보유자가 생존 상태이고, 생존자 중 최고패가 **팔땡 이하**일 때 자동 트리거

- **D-07 재경기 우선순위 (충돌 해소):**
  - 일반 구사 + 동점 재경기: **구사 재경기 우선** — 알리 이하에서는 동점 재경기 발생 불가
  - 멍텅구리구사 + 동점 재경기: **멍텅구리구사 재경기 우선** — 팔땡 이하에서는 동점 재경기 발생 불가
  - **암행어사 엣지케이스:** 암행어사로 일팔/일삼광땡을 이긴 경우 → 구사 패 무시, 재경기 없음 (암행어사 승리가 최종)
  - **땡잡이 엣지케이스:**
    - 땡잡이로 승리 + 일반 구사 존재 → 일반 구사 재경기 **없음** (땡잡이가 구땡 이하 모든 땡을 이기므로)
    - 땡잡이로 승리 + **멍텅구리구사** 존재 → **멍텅구리구사 재경기 트리거됨** (땡잡이 승리는 팔땡 이하 조건을 충족하므로)

- **D-08 재참여 UX:** 재경기 트리거 시 다이한 플레이어에게 **15초 타임아웃 모달** 표시
  - 모달에 판돈의 절반 금액 명시 (예: "판돈 10,000원의 절반 = 5,000원을 내고 재참여")
  - 승낙 시: 해당 플레이어 잔액에서 즉시 차감, 판돈(pot)에 증액
  - 거절 또는 타임아웃: 해당 플레이어 불참 처리 (생존자들끼리만 재경기 진행)

- **D-09 재경기 FSM:** 재경기는 기존 `rematch-pending` phase 패턴(HAND-07, 동점 재경기)을 참고하여 별도 phase 추가.
  - `'gusa-pending'` — 94재경기 대기 상태, 다이 플레이어 재참여 결정 수집
  - 모든 대상 플레이어 결정 완료 → 새 딜로 전환 (앤티 없음, pot 유지)
  - 재경기 참여자: (기존 생존자) + (재참여 선택한 다이 플레이어)
  - 재경기 선 결정: 기존 게임의 선 플레이어 유지 (또는 planner 재량)

- **D-10 재경기 중 칩 부족:** 재참여 결제 시 잔액이 판돈 절반 미만인 경우 처리 → planner 재량 (거절 처리 또는 전액 가능 금액으로 처리)

### Claude's Discretion

- `gusa-pending` phase에서 15초 타임아웃 구현 방식 (서버 setTimeout vs 클라이언트 카운트다운)
- 재경기 선 결정 상세 (기존 선 유지 or 새 선 결정)
- 재참여 결제 시 칩 부족 플레이어 처리 (거절 처리 권장)
- 땡값 UI 컴포넌트 배치 (기존 result 화면 확장 vs 별도 섹션)

</decisions>
