## 미완료: Plan 02 Task 2 수동 검증 통과 (+ 하위 버그 수정)

사용자가 발견한 버그 목록 (우선순위 순):

### A. result 화면 땡값 표시 방식 변경
**현재**: 모든 플레이어카드 밖에 별도 땡값 섹션 표시
**원하는 것**:
- 승리자 카드 하단: `판돈+땡값 합산 +N원` + 그 아래 `땡값 +N원`
- 패배자(다이) 카드 하단: `-(베팅잃은돈+땡값낸돈) 합산` + 그 아래 `땡값 -N원`
파일: `packages/client/src/components/layout/ResultScreen.tsx`

### B. 손패 패널 특수패 이름 표기 오류
- 삼칠(3+7 normal): 0끗이 아닌 "망통" 표기
- 아홉끗: "갑오" 표기
- 열끗47(4+7 yeolkkeut): 한끗이 아닌 "암행어사" 표기
- 3+7 normal: "땡잡이" 표기
- 4+9 normal: "구사" 표기
- 4+9 yeolkkeut: "멍텅구리구사" 표기
파일: 손패 표시 컴포넌트 (HandPanel 또는 유사 컴포넌트, 정확한 경로 확인 필요)

### C. 구사 재경기 전원 생존 시 안내 모달 미표시
**현재**: 전원 생존이면 즉시 shuffling (안내 없음)
**원하는 것**: "구사라서 재경기합니다" 안내 모달 표시 후 shuffling 진행
(다이 플레이어가 있을 때는 gusa-pending 재참여 모달이 있으므로 해당 없음)

### D. 구사 재경기에서 기리(cutting) 제거
**현재**: _startGusaRematch()가 phase='shuffling'으로 전환 → 기리(cutting) 포함
**원하는 것**: 구사 재경기는 선플레이어 섞기(shuffling)만 있고 기리 없이 바로 dealing
파일: `packages/server/src/game-engine.ts` `_startGusaRematch()` 수정

### E. 재경기(구사/동점)에서 땡값 면제
**현재**: 재경기 후 발생한 땡에도 땡값 적용됨
**원하는 것**: `isRematch` 또는 `roundType` 플래그로 재경기 라운드 판별 → _settleTtaengValue에서 면제
파일: `packages/server/src/game-engine.ts`

### F. 세장섯다 카드 선택 타이밍 버그 (핵심 버그)
**현재**: 카드 선택이 2차 베팅 전에 발생
**원하는 것**: 2차 베팅 완료 후 카드 선택 단계
FSM 순서: dealing → betting-1 → card-select → betting-2 → showdown/result
vs 원하는 순서: dealing → betting-1 → betting-2 → card-select → showdown/result
파일: `packages/server/src/game-engine.ts` 세장섯다 FSM 수정

### G. 세장섯다 선택 카드가 족보 판정에 미반영

**원하는 것**: selectedCards[0], selectedCards[1]로 족보 판정
파일: `packages/server/src/game-engine.ts` `_resolveShowdownSejang()` 수정

### H. 세장섯다 잃은돈 합계 계산 오류
