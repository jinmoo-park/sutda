## Copywriting Contract

### GusaRejoinModal (신규 컴포넌트)

| 요소 | 카피 |
|------|------|
| DialogTitle | `94재경기 — 재참여 여부를 선택하세요` |
| DialogDescription (일반 구사) | `구사(4+9)로 재경기가 발생했습니다. 판돈 {N}원의 절반인 {N/2}원을 내고 재참여할 수 있습니다.` |
| DialogDescription (멍텅구리구사) | `멍텅구리구사로 재경기가 발생했습니다. 판돈 {N}원의 절반인 {N/2}원을 내고 재참여할 수 있습니다.` |
| 타임아웃 안내 | `{N}초 안에 결정하지 않으면 불참 처리됩니다.` |
| 재참여 확인 버튼 | `재참여 ({N/2}원 납부)` |
| 거절 버튼 | `불참` |
| 칩 부족 시 재참여 버튼 비활성화 안내 | `칩이 부족합니다` (버튼 disabled + 툴팁 or 설명 텍스트) |
| 생존자 안내 문구 | `다른 플레이어의 재참여 결정을 기다리는 중입니다...` |

### ResultScreen 땡값 섹션 (기존 컴포넌트 확장)

| 요소 | 카피 |
|------|------|
| 섹션 헤딩 | `땡값 정산` |

## Interaction Contracts

### 타임아웃 카운트다운 (GusaRejoinModal)

- **구현 방식:** 클라이언트 카운트다운 (`useState` + `useEffect` 1초 interval) — `RechargeVoteModal` 방식과 다름 (recharge는 타임아웃 없음)
- **초기값:** 15초 (`gusa-pending` phase 진입 시 카운트다운 시작)
- **0초 도달 시:** 자동으로 `gusa-rejoin` 이벤트에 `{ join: false }` emit
- **버튼 클릭 시:** 즉시 emit + 모달 닫힘 (카운트다운 중단)
- **카운트다운 표시:** `{N}초 안에 결정하지 않으면 불참 처리됩니다.` — N은 실시간 업데이트
- **Accent 색상 적용:** 카운트다운 숫자 강조 (`text-primary font-semibold`)

### 재참여 버튼 비활성화 조건

- 플레이어 칩 잔액 < 판돈 절반: `Button disabled` + `칩이 부족합니다` 설명 텍스트
- 이미 결정한 경우: 버튼 비활성화 + "결정 완료" 표시

### 땡값 배지 색상 규칙

- 납부 플레이어 (음수): `bg-red-600 text-white` (기존 `ResultScreen` chipDelta 패턴 동일)
- 수령 플레이어 (양수): `bg-green-600 text-white` (기존 `ResultScreen` chipDelta 패턴 동일)
