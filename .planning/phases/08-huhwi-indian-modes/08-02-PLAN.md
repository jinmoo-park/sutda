**B. handleGameAction을 async로 변경하고 per-player emit 적용** (per D-13):

현재 동기 함수를 async로 변환하고, 성공 브로드캐스트를 항상 per-player emit으로 처리한다. 단순화 방식 채택 — 모드 분기 없이 항상 getStateFor를 호출:

```typescript
async function handleGameAction(
  socket: TypedSocket,
  roomId: string,
  action: () => void
): Promise<void> {
  try {
    action();
    const engine = getEngine(roomId);

    // 항상 per-player emit — getStateFor가 인디언 모드에서만 마스킹 적용
    // 다른 모드에서는 getState()와 동일한 상태 반환 (성능 동등)
    const sockets = await io.in(roomId).fetchSockets();
    for (const s of sockets) {
      const pid = s.data.playerId;
      s.emit('game-state', engine.getStateFor(pid) as GameState);
    }
  } catch (err: any) {
    socket.emit('game-error', {
      code: err.message || 'UNKNOWN_ERROR',
      message: ERROR_MESSAGES[err.message] || err.message || '알 수 없는 오류',
    });
  }
}
```

이 변경으로 기존 모든 handleGameAction 호출이 자동으로 per-player emit을 사용한다.

주의: handleGameAction 호출부에서 await가 필요하지 않음 (fire-and-forget 패턴 유지). TypedSocket 타입에서 playerId가 socket.data.playerId로 접근 가능한지 확인. 없으면 `s.data?.playerId` 사용.

**D. 인디언 betting-1 완료 후 dealing-extra 자동 처리**:

기존 'bet-action' 핸들러를 찾아서 인디언 모드 분기를 추가한다. bet-action 처리 후 phase가 'dealing-extra'이면 즉시 dealExtraCardIndian()을 호출:

기존 bet-action 핸들러(handleGameAction 내부 or 직후)에서:
```typescript
socket.on('bet-action', ({ roomId, action }) => {
  const playerId = socket.data.playerId;
  handleGameAction(socket, roomId, () => {
    const engine = getEngine(roomId);
    engine.placeBet(playerId, action);  // 기존 메서드명 확인 후 사용

    // 인디언 모드: betting-1 종료 후 dealing-extra 자동 처리
    const state = engine.getState();
    if (state.mode === 'indian' && state.phase === 'dealing-extra') {
      engine.dealExtraCardIndian();
    }
  });
});
```
