**단계 1: 2GB 스왑 파일 생성 (1GB RAM OOM 방지)**

```bash
sudo swapon /swapfile
```


**단계 6: iptables 방화벽 규칙 추가**

주의: Oracle Cloud Ubuntu 이미지는 INPUT 체인 끝에 REJECT ALL이 있으므로 반드시 `-I`(insert)를 사용한다. `-A`(append) 금지.

```bash
sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 80 -j ACCEPT
sudo iptables -I INPUT 7 -m state --state NEW -p tcp --dport 443 -j ACCEPT
sudo iptables -I INPUT 8 -m state --state NEW -p tcp --dport 3000 -j ACCEPT
sudo apt-get install -y iptables-persistent
sudo netfilter-persistent save
```

OCI 콘솔에서 VCN Security List에 80/443 포트도 열려 있어야 한다 (user_setup에 기재됨).

**단계 7: DuckDNS 설정**

사용자가 duckdns.org에서 서브도메인과 토큰을 확인한 후 진행.
서브도메인 예시: `sutda-game` (실제 값은 사용자 결정 — Claude's Discretion)

```bash
mkdir -p ~/duckdns && cd ~/duckdns
cat > duck.sh << 'DUCKEOF'
#!/bin/bash
echo url="https://www.duckdns.org/update?domains=SUBDOMAIN_HERE&token=TOKEN_HERE&ip=" | curl -k -o ~/duckdns/duck.log -K -
DUCKEOF
chmod 700 duck.sh
```

사용자가 제공하는 SUBDOMAIN과 TOKEN 값으로 duck.sh의 플레이스홀더를 교체한다.

```bash
./duck.sh
cat duck.log  # "OK" 확인

# crontab 등록 (5분마다)
(crontab -l 2>/dev/null; echo "*/5 * * * * ~/duckdns/duck.sh >/dev/null 2>&1") | crontab -
```

**단계 2: nginx 설정 파일 생성 (HTTP 전용 — certbot 실행 전)**

`/etc/nginx/sites-available/sutda` 파일 생성:
```nginx
upstream sutda_backend {
  server 127.0.0.1:3001;
  keepalive 64;
}

server {
  listen 80;
  server_name SUBDOMAIN.duckdns.org;

  location /.well-known/acme-challenge/ {
    root /var/www/html;
  }

  location / {
    proxy_pass         http://sutda_backend;
    proxy_http_version 1.1;

    proxy_set_header   Upgrade    $http_upgrade;
    proxy_set_header   Connection "upgrade";

    proxy_set_header   Host              $host;
    proxy_set_header   X-Real-IP         $remote_addr;
    proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
    proxy_set_header   X-Forwarded-Proto $scheme;

    proxy_read_timeout  86400s;
    proxy_send_timeout  86400s;
    proxy_connect_timeout 10s;

    proxy_buffering off;
  }
}
```

SUBDOMAIN은 Task 1에서 설정한 DuckDNS 서브도메인으로 교체.

```bash
# default 사이트 비활성화 + sutda 활성화

sudo ln -sf /etc/nginx/sites-available/sutda /etc/nginx/sites-enabled/sutda
sudo nginx -t  # syntax OK 확인
sudo systemctl reload nginx
```

**단계 4: certbot SSL 인증서 발급**
```bash
sudo certbot --nginx -d SUBDOMAIN.duckdns.org
```

certbot이 nginx 설정을 자동으로 수정하여 HTTPS 리다이렉트 + SSL 설정을 추가한다.
certbot 실행 후 nginx 설정에 `listen 443 ssl`, `ssl_certificate` 등이 자동 추가됨.

nginx에 certbot이 추가한 HTTPS 블록의 `proxy_read_timeout`이 유지되는지 확인.
만약 certbot이 새 server 블록을 생성했다면, WebSocket 관련 헤더가 누락될 수 있으므로
`/etc/nginx/sites-available/sutda`를 확인하고 필요시 다음을 443 블록에 추가:

```nginx
# HTTPS 블록에 WebSocket 헤더가 없으면 추가
proxy_set_header   Upgrade    $http_upgrade;
proxy_set_header   Connection "upgrade";
proxy_read_timeout  86400s;
proxy_send_timeout  86400s;
proxy_buffering off;
```

```bash
sudo nginx -t
sudo systemctl reload nginx
```

HTTPS 확인:
```bash
curl -s -o /dev/null -w "%{http_code}" https://SUBDOMAIN.duckdns.org/
# 200 확인
```
