2. **CORS fallback 수정 (D-04):**
   `packages/server/src/index.ts` 라인 57의 CORS origin 설정을 변경한다:
   ```typescript

   origin: process.env.CLIENT_ORIGIN || 'https://sutda.duckdns.org',
   ```
   `true`(모든 origin 허용) 대신 명시적 프로덕션 도메인으로 fallback. 개발 환경에서는 `CLIENT_ORIGIN=http://localhost:5173` 등으로 환경변수 설정.

3. **Rate Limiting 미들웨어 추가 (D-07):**
   파일 상단에 import 추가:
   ```typescript
   import { RateLimiterMemory } from 'rate-limiter-flexible';
   ```

   `io.on('connection')` 콜백 바로 앞 (라인 143 직전)에 limiter 인스턴스 생성:
   ```typescript
   const socketEventRateLimiter = new RateLimiterMemory({
     points: 20,
     duration: 1,
   });
   ```

   `io.on('connection', (socket) => {` 콜백 내부 최상단 (라인 144, create-room 핸들러 이전)에 socket.use 미들웨어 삽입:
   ```typescript
   socket.use(async ([event, ...args], next) => {
     try {
       await socketEventRateLimiter.consume(socket.id);
       next();
     } catch {
       // rate limit 초과: 이벤트 무시, 연결 유지
       // next()를 호출하지 않으면 이벤트 핸들러 실행 안 됨
     }
   });
   ```

   **주의:** `next(new Error(...))` 호출하면 연결이 끊어진다 (Pitfall 3). `next()`를 호출하지 않는 방식으로 이벤트를 무시해야 한다.

4. **send-chat roomId 검증 추가 (A01):**
   `send-chat` 핸들러(라인 286) 내부 최상단에 검증 추가:
   ```typescript
   socket.on('send-chat', ({ roomId, text }) => {
     // A01: 요청자가 해당 방에 속하는지 검증
     if (socket.data.roomId !== roomId) return;
     if (!text || text.trim().length === 0) return;
     // ... 기존 코드
   });
   ```
   `socket.data.roomId`는 `create-room`/`join-room` 시 설정되므로 이 검증으로 다른 방에 채팅 전송하는 것을 차단한다.

```typescript
describe('보안: rate limiting (D-07)', () => {
  afterEach(() => { /* cleanup clients */ });

  it('소켓당 초당 20개 이벤트 초과 시 일부 이벤트가 무시된다', async () => {
    const client = createClient();
    await new Promise<void>((resolve) => client.on('connect', resolve));

    // 방 생성
    client.emit('create-room', { nickname: 'rate-test', initialChips: 100000 });
    const { roomId } = await waitForEvent<{ roomId: string }>(client, 'room-created');

    // 30개 send-chat 이벤트를 동기적으로 빠르게 전송 (20개 초과)
    const chatPromises: Promise<any>[] = [];
    const received: any[] = [];
    client.on('chat-message', (msg) => received.push(msg));

    for (let i = 0; i < 30; i++) {
      client.emit('send-chat', { roomId, text: `msg-${i}` });
    }

    // 충분한 대기 시간 후 수신된 메시지 수가 30개 미만인지 확인
    await new Promise((r) => setTimeout(r, 500));
    expect(received.length).toBeLessThan(30);
    // 연결은 유지되어야 한다
    expect(client.connected).toBe(true);

    client.disconnect();
  });
});

describe('보안: send-chat 접근 제어 (A01)', () => {
  it('자신이 속하지 않은 방에 채팅을 보내면 무시된다', async () => {
    const client1 = createClient();
    const client2 = createClient();
    await Promise.all([
      new Promise<void>((resolve) => client1.on('connect', resolve)),
      new Promise<void>((resolve) => client2.on('connect', resolve)),
    ]);

    // client1이 방 생성
    client1.emit('create-room', { nickname: 'owner', initialChips: 100000 });
    const { roomId } = await waitForEvent<{ roomId: string }>(client1, 'room-created');

    // client2는 다른 방을 생성 (자신의 roomId가 다름)
    client2.emit('create-room', { nickname: 'other', initialChips: 100000 });
    await waitForEvent<{ roomId: string }>(client2, 'room-created');

    // client2가 client1의 roomId로 채팅 시도
    const received: any[] = [];
    client1.on('chat-message', (msg) => received.push(msg));

    client2.emit('send-chat', { roomId, text: 'injection attempt' });
    await new Promise((r) => setTimeout(r, 300));

    // client1의 방에 메시지가 도달하지 않아야 한다
    expect(received.length).toBe(0);

    client1.disconnect();
    client2.disconnect();
  });
});
```

**주의:** rate limiting 테스트는 타이밍에 민감할 수 있다. 30개 이벤트를 동기적으로 전송하고 수신 개수가 30 미만임을 확인하는 방식으로 검증한다. 정확한 수를 단언하지 않고 `toBeLessThan(30)`으로 유연하게 처리한다.
