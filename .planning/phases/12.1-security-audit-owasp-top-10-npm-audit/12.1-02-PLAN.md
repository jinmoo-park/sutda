사용자가 Oracle VM에 SSH 접속하여 nginx 설정을 수정해야 한다. Claude는 정확한 설정 내용을 제공한다.

**Step 1: 현재 nginx 설정 확인**
```bash
ssh opc@sutda.duckdns.org
sudo cat /etc/nginx/sites-available/sutda
```

**Step 2: http{} 블록에 limit_conn_zone 추가**
`/etc/nginx/nginx.conf` 또는 `/etc/nginx/conf.d/` 내 적절한 위치의 `http {}` 블록에 추가:
```nginx
limit_conn_zone $binary_remote_addr zone=sutda_ws_conn:10m;
```

**Step 3: server{} 블록에 보안 헤더 추가 (HTTPS server 블록)**
`/etc/nginx/sites-available/sutda` 파일의 HTTPS server {} 블록 내부에 추가:
```nginx
# 보안 헤더 (D-05)
add_header X-Content-Type-Options "nosniff" always;
add_header X-Frame-Options "SAMEORIGIN" always;
add_header Content-Security-Policy "default-src 'self'; connect-src 'self' wss://sutda.duckdns.org; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data:;" always;
```

**Step 4: location /socket.io/ 블록에 limit_conn 추가**
기존 Socket.IO 프록시 location 블록에 `limit_conn` 추가:
```nginx
location /socket.io/ {
    limit_conn sutda_ws_conn 5;
    proxy_pass http://localhost:3001;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_read_timeout 86400;
}
```

**Step 5: 설정 검증 및 재시작**
```bash
sudo nginx -t
sudo systemctl reload nginx
```

**Step 6: 헤더 확인**
```bash
curl -I https://sutda.duckdns.org
```
응답에 `X-Content-Type-Options: nosniff`, `X-Frame-Options: SAMEORIGIN`, `Content-Security-Policy: ...` 헤더가 포함되어야 한다.

**Step 7: VM CLIENT_ORIGIN 환경변수 확인 (D-04 보조)**
```bash
pm2 env 0 | grep CLIENT_ORIGIN
```
미설정이면:
```bash
pm2 set sutda:CLIENT_ORIGIN https://sutda.duckdns.org
# 또는 ecosystem.config.cjs에 env 추가 후 pm2 restart
```

## OWASP Top 10 항목별 상세

### A01: Broken Access Control
- **관련성:** HIGH
- **현황:** socket.data.playerId 서버 권위 모델 준수. canStartGame() 방장 검증 존재.
- **발견 이슈 (Medium):** `send-chat` 핸들러에서 roomId 검증 없음 — 타 방 채팅 전송 가능
- **조치:** socket.data.roomId !== roomId 검증 추가 완료 (Plan 01)
- **잔여 Medium:** 기타 이벤트 핸들러의 roomId 교차 검증은 socket.rooms.has() 패턴으로 이미 보호 (Socket.IO room 구독 메커니즘)

### A02: Cryptographic Failures
- **관련성:** HIGH
- **발견 이슈 (Critical):** `.gitignore`에 `.env` 누락 — ROOM_CREATE_PASSWORD 등 시크릿 노출 위험
- **조치:** `.gitignore`에 `.env` 추가 완료 (commit 254f81c)
- **현황:** .env는 untracked 상태 유지 확인

### A03: Injection (XSS)
- **관련성:** MEDIUM
- **현황:** React JSX 렌더링이 기본 XSS escape 제공. 채팅 텍스트 200자 제한 존재.
- **발견 이슈 (Medium):** `packages/client/src/main.tsx` 에러 핸들러에서 `document.body.innerHTML` 직접 삽입 — 오염된 에러 메시지의 XSS 가능성 (공격 경로 좁음)
- **조치:** 보고서 기록 (D-06 결정). 별도 페이즈에서 textContent 전환 검토.

### A04: Insecure Design — 비해당
- 회원가입 없는 닉네임 기반 구조는 설계 의도. 보안 설계 결함 아님.

### A05: Security Misconfiguration
- **관련성:** HIGH
- **발견 이슈 (Critical):** CORS fallback이 `true` (모든 origin 허용)
- **조치:** fallback을 `'https://sutda.duckdns.org'`로 변경 완료 (Plan 01)
- **발견 이슈 (High):** HTTP 보안 헤더 없음 + rate limiting 없음
- **조치:**
  - nginx 보안 헤더 3종 추가 완료 (Plan 02 Task 1)
  - Socket.IO socket.use() rate limiting 미들웨어 추가 완료 (Plan 01)
  - nginx limit_conn IP당 5개 연결 제한 추가 완료 (Plan 02 Task 1)

### A06: Vulnerable and Outdated Components — 통과
- `pnpm audit` 결과 취약점 없음 (D-02)

### A07: Identification and Authentication Failures — 비해당
- 회원가입/로그인 없음. 닉네임 기반 세션은 설계 의도.

### A08: Software and Data Integrity Failures — 비해당
- pnpm-lock.yaml 존재. GitHub public repo. CI/CD 파이프라인 없음 (수동 배포).

### A09: Security Logging and Monitoring Failures
- **관련성:** LOW
- **발견 이슈 (Low):** PM2 로그 외 별도 에러 로깅/모니터링 없음
- **조치:** 보고서 기록. 사용 규모 고려 시 현재 수준 적정. 향후 필요 시 Winston/Pino 도입 검토.

### A10: Server-Side Request Forgery (SSRF) — 비해당
- 서버가 외부 URL 요청을 수행하지 않음.
