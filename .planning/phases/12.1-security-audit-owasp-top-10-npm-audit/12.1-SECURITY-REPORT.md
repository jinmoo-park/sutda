## OWASP Top 10 항목별 상세

### A01: Broken Access Control

**관련성:** HIGH
**전체 현황:** 서버 권위 모델(`socket.data.playerId` 기반 게임 액션 처리) 준수. 방장 검증(`canStartGame()`) 존재.

#### 발견 이슈

| 이슈 | 심각도 | 상태 |
|------|--------|------|
| `send-chat` 핸들러에서 `roomId` 교차 검증 없음 — 타 방 채팅 전송 가능 | Medium | **수정 완료** |

**이슈 상세 (send-chat roomId 검증 누락):**
- **위치:** `packages/server/src/index.ts` — `send-chat` 이벤트 핸들러
- **취약점:** 클라이언트가 자신이 속하지 않은 `roomId`를 전달하여 타 방에 채팅 전송 가능
- **조치:** `socket.data.roomId !== roomId` 서버 사이드 검증 추가 (Plan 01 — commit `bde0a48`)
- **검증:** 보안 통합 테스트 추가 및 통과 확인 (`describe('보안: send-chat 접근 제어 (A01)')`)

**기타 이벤트 핸들러 접근 제어:**
- `start-game`, `select-mode`: `canStartGame(roomId, socket.id)` 패턴으로 방장 검증 (OK)
- `bet-action`, `select-dealer-card` 등 게임 액션: `socket.data.playerId`로 전달 — 서버 권위 원칙 준수 (OK)
- 기타 방 관련 이벤트: `socket.rooms.has(roomId)` (Socket.IO 내장 room 구독 메커니즘)으로 간접 보호 (OK)

### A03: Injection (XSS)

**관련성:** MEDIUM

#### 발견 이슈

| 이슈 | 심각도 | 상태 |
|------|--------|------|
| `main.tsx` 에러 핸들러에서 `document.body.innerHTML` 직접 삽입 | Medium | 보고서 기록 (D-06) |
| 닉네임 길이 제한 없음 (서버 사이드) | Medium | 보고서 기록 (D-06) |

**이슈 상세 (innerHTML XSS):**
- **위치:** `packages/client/src/main.tsx` — `window.onerror` 및 `window.onunhandledrejection` 핸들러
- **취약점:** `document.body.innerHTML = \`<pre ...>${msg}\n${err?.stack}\`</pre>`` — 오염된 에러 메시지로 인한 XSS 가능성
- **공격 경로:** 외부 스크립트 주입으로 오염된 `msg` 또는 `err.stack`이 실행될 수 있으나, 이 에러 핸들러는 다른 스크립트 오류가 없는 한 실행되지 않아 공격 경로가 좁음
- **D-06 결정:** 보고서 기록만. 향후 별도 작업에서 `textContent` 전환 권장.

**기타 XSS 보호 현황:**
- React JSX 렌더링: 기본 HTML escape 제공 (OK)
- 채팅 텍스트: `text.trim().slice(0, 200)` — 200자 제한 존재 (OK)
- nginx CSP: `Content-Security-Policy` 헤더 추가 완료 (Plan 02 Task 1) — 브라우저 레벨 XSS 방어 강화

**SQL Injection:** 데이터베이스 없음 (인메모리 Map) — 비해당.

### A05: Security Misconfiguration

**관련성:** HIGH

#### 발견 이슈

| 이슈 | 심각도 | 상태 |
|------|--------|------|
| CORS fallback `origin: true` — `CLIENT_ORIGIN` 미설정 시 모든 origin 허용 | **Critical** | **수정 완료** |
| HTTP 보안 헤더 없음 (`X-Content-Type-Options`, `X-Frame-Options`, `CSP`) | High | **수정 완료** |
| Rate limiting 없음 — 이벤트 폭탄/연결 폭풍 취약 | High | **수정 완료** |

**이슈 상세 (CORS fallback):**
- **위치:** `packages/server/src/index.ts` — Socket.IO CORS 설정
- **취약점:** `origin: process.env.CLIENT_ORIGIN || true` — `CLIENT_ORIGIN` 미설정 시 `Access-Control-Allow-Origin: *` 동작
- **조치:** fallback을 `'https://sutda.duckdns.org'`로 변경 (Plan 01 — commit `bde0a48`)
- **검증:** 코드 리뷰 및 통합 테스트 확인

**이슈 상세 (HTTP 보안 헤더 — nginx):**
- **위치:** `/etc/nginx/sites-available/sutda` (Oracle VM)
- **취약점:** `X-Content-Type-Options`, `X-Frame-Options`, `Content-Security-Policy` 헤더 없음
- **조치:** nginx `add_header` 지시자로 3종 헤더 추가 (Plan 02 Task 1 — 사용자 적용)
  ```
  add_header X-Content-Type-Options "nosniff" always;
  add_header X-Frame-Options "SAMEORIGIN" always;
  add_header Content-Security-Policy "default-src 'self'; connect-src 'self' wss://sutda.duckdns.org; ..." always;
  ```
- **검증:** `curl -I https://sutda.duckdns.org` — 3개 헤더 모두 확인됨

**이슈 상세 (Rate Limiting):**
- **위치:** Oracle VM — 서버 미들웨어 + nginx 레이어
- **취약점:** 이벤트 폭탄 / 연결 폭풍 방어 없음 (Oracle VM 1GB RAM 환경)
- **조치 (2레이어 방어):**
  - **nginx 레이어:** `limit_conn_zone` + `limit_conn sutda_ws_conn 5` — IP당 WebSocket 동시 연결 최대 5개 (Plan 02 Task 1)
  - **서버 미들웨어 레이어:** `rate-limiter-flexible` `RateLimiterMemory` — 소켓당 초당 20개 이벤트 제한 (`socket.use()` 패턴) (Plan 01 — commit `bde0a48`)
- **검증:** 보안 통합 테스트 추가 및 통과 (`describe('보안: rate limiting (D-07)')`)

회원가입/로그인 없음. 닉네임 기반 세션은 설계 의도 (D-04). 게임 서비스 특성상 강력한 인증이 필요하지 않은 소규모 실시간 게임이다.

### A09: Security Logging and Monitoring Failures

**관련성:** LOW

#### 발견 이슈

| 이슈 | 심각도 | 상태 |
|------|--------|------|
| PM2 기본 로그 외 별도 에러 로깅/모니터링 없음 | Low | 보고서 기록 |

**현황:**
- 에러 로깅: `console.error()` 출력만 존재 — PM2가 `~/.pm2/logs/` 에 파일로 기록
- 구조화 로깅 없음 — 에러 추적 및 분석 어려움
- 보안 이벤트 로깅 없음 — rate limit 초과, 접근 거부 이벤트 기록 없음
- 모니터링 대시보드 없음

**평가:** 현재 서비스 규모(소규모 개인 게임 서버) 고려 시 PM2 기본 로그 수준은 적정. 향후 필요 시 Winston/Pino 등 구조화 로깅 도입 검토 권장.

### A10: Server-Side Request Forgery (SSRF) — 비해당

서버가 외부 URL 요청을 수행하지 않음. `fetch()`, `axios`, HTTP 클라이언트 라이브러리 미사용. 서버는 오직 Socket.IO 이벤트 처리 및 정적 파일 서빙만 수행.
