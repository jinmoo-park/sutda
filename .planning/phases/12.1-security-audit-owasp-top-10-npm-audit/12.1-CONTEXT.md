<decisions>
## Implementation Decisions

### 수정 범위 (D-01)
- **Critical/High 취약점**: 감사와 동일 페이즈에서 즉시 수정 (report + fix)
- **Medium/Low 취약점**: 감사 보고서에 기록만, 수정은 별도

### 사전 스캔 결과 (D-02)
- `pnpm audit` 결과: **취약점 없음** — 의존성 스캔 통과. 추가 npm audit 작업 불필요.
- 실질적인 보안 이슈는 코드 레벨과 인프라 설정에 집중.

### `.env` gitignore 누락 → Critical (D-03)
- 현재 `.gitignore`에 `.env.local`, `.env*.local`만 있고 `.env`가 없음
- `.env` 파일에 `STITCH_ACCESS_TOKEN`, `GOOGLE_CLOUD_PROJECT` 포함 — 실수로 커밋 시 시크릿 노출
- **즉시 수정**: `.gitignore`에 `.env` 추가

### CORS 설정 점검 → Critical (D-04)
- 현재 코드: `origin: process.env.CLIENT_ORIGIN || true`
- `CLIENT_ORIGIN` 미설정 시 모든 origin 허용 — 프로덕션 VM 환경변수 확인 필수
- **즉시 수정**: Oracle VM(`sutda.duckdns.org` 서버)에서 `CLIENT_ORIGIN=https://sutda.duckdns.org` 설정 여부 확인. 미설정 시 환경변수 추가.
- 코드 레벨 fallback도 `true` 대신 명시적 origin 문자열로 변경 검토

### HTTP 보안 헤더 → nginx 설정으로 추가 (D-05)
- helmet.js 같은 Node.js 미들웨어 추가하지 않음
- nginx 설정(`/etc/nginx/sites-available/sutda` 등)에 응답 헤더로 추가:
  - `X-Content-Type-Options: nosniff`
  - `X-Frame-Options: SAMEORIGIN`
  - `Content-Security-Policy: default-src 'self'; connect-src 'self' wss://sutda.duckdns.org; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data:;`
- HSTS는 nginx가 이미 Let's Encrypt + TLS로 처리 중

### 입력값 검증 (D-06)
- 닉네임 길이 제한 없음, 채팅 텍스트 길이 제한 없음 → Medium
- **보고서에 기록**, 즉시 수정 대상 아님 (React JSX 렌더링이 기본 XSS escape 제공)
- 필요 시 별도 페이즈에서 서버사이드 validation 강화

### Rate Limiting (D-07) ← deferred에서 즉시 수정으로 격상
- 서버가 공개 인터넷에 노출되어 있고 Oracle VM은 1GB RAM — 자동 스캐너/연결 폭풍으로 서비스 중단 위험
- **2레이어 방어 구현:**
  - **nginx 레이어**: IP당 WebSocket 연결 최대 5개 제한 (nginx 설정 파일 수정)
  - **서버 미들웨어 레이어**: Socket.IO 미들웨어로 소켓당 초당 20개 이벤트 제한 (in-memory 카운터)
- 한계 초과 시 연결 차단 또는 이벤트 무시 처리

### OWASP Top 10 감사 범위 (D-08)
- 이 프로젝트에 relevant한 항목만 집중 점검:
  - A01 (Broken Access Control) — 방 입장/게임 액션 권한 검증
  - A02 (Cryptographic Failures) — 시크릿/환경변수 노출
  - A03 (Injection) — 닉네임/채팅 입력 XSS 가능성
  - A05 (Security Misconfiguration) — CORS, HTTP 헤더, nginx 설정
  - A06 (Vulnerable Components) — npm 의존성 (이미 통과)
- 비해당 항목 (SQL, 인증/세션 관리, 암호화 등) — 기록 후 skip

</decisions>
