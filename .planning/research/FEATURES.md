# Feature Research

**Domain:** 실시간 멀티플레이어 웹 카드 게임 (온라인 섯다)
**Researched:** 2026-03-29
**Confidence:** MEDIUM (웹 검색 불가, 훈련 데이터 + rule_draft.md 기반 분석)

## Feature Landscape

### Table Stakes (사용자가 당연히 기대하는 기능)

이 기능이 없으면 제품이 "미완성"으로 느껴진다. 크레딧을 주지 않지만, 없으면 즉시 이탈한다.

| Feature | 왜 기대하는가 | 복잡도 | 비고 |
|---------|--------------|--------|------|
| **실시간 게임 상태 동기화** | 카드 배분, 베팅, 결과가 모든 플레이어에게 즉시 반영되어야 함 | HIGH | WebSocket 기반, 서버 권위(authoritative) 모델 필수 |
| **방 생성 및 링크 초대** | "링크 하나로 친구들과 플레이"가 핵심 가치 | MEDIUM | 방 코드 포함 URL, 닉네임 입력 후 즉시 입장 |
| **정확한 족보 판정** | 섯다의 본질. 틀리면 게임 자체가 무의미 | HIGH | 광땡 3종 > 땡 10종 > 특수패 6종 > 끗 계산, 구사/멍텅구리구사 재경기 트리거 |
| **베팅 시스템 (콜/레이즈/다이)** | 카드 게임에서 베팅은 핵심 인터랙션 | MEDIUM | 자유 금액 레이즈, 턴 기반 순서 관리 |
| **칩/포인트 추적** | 누가 얼마 가지고 있는지 실시간 표시 필수 | LOW | 만원 단위, 시작 금액 설정, 소진 시 재충전 |
| **턴 관리 및 타이머** | 누구 차례인지 명확해야 함. 무한 대기 방지 필수 | MEDIUM | 턴 하이라이트 + 시간 제한 (30~60초) |
| **카드 셔플 및 배분 애니메이션** | 카드 게임의 기본 시각 피드백 (와이어프레임이라도 움직임 필요) | MEDIUM | 와이어프레임에서도 카드가 이동하는 기본 트랜지션 |
| **원형 테이블 레이아웃** | 2~6인이 둘러앉은 형태, 각자 카드/칩 영역 표시 | MEDIUM | 반응형 (2인~6인 동적 배치) |
| **게임 결과 화면** | 판 종료 시 승자, 각자 패, 획득/손실 금액 명확히 표시 | LOW | 모든 카드 공개 + 족보 이름 표시 |
| **연결 끊김 처리** | 친구가 와이파이 끊겨도 게임이 깨지지 않아야 함 | MEDIUM | 자동 재연결 + 일정 시간 후 자동 다이 처리 |
| **5가지 게임 모드** | rule_draft에 정의된 하우스룰 전체가 프로젝트 범위 | HIGH | 오리지날/세장섯다/한장공유/후회의섯다/인디언섯다 |
| **선 결정 (밤일낮장)** | 첫 판 시작의 공정성, 하우스룰의 일부 | LOW | 서울시간 기준 시간대별 로직 |
| **기리/퉁 메커니즘** | 셔플 후 카드 분배 방식의 핵심, 하우스룰 일부 | MEDIUM | 왼쪽 플레이어 선택 UI 필요 |

### Differentiators (차별화 기능)

필수는 아니지만 사용자 경험을 크게 향상시키는 기능들.

| Feature | 가치 제안 | 복잡도 | 비고 |
|---------|----------|--------|------|
| **땡값 자동 정산** | 오리지날 모드에서 다이한 땡 보유자 -> 승자 자동 지불. 수동 계산 불필요 | LOW | 판정 로직에 포함시키면 됨. 규칙이 명확 |
| **94재경기 자동 트리거** | 구사 발생 시 조건 판단 + 죽은 플레이어 재참여 선택 UI | MEDIUM | 판돈 절반 계산 + 재참여 확인 UX 설계 필요 |
| **게임 내 실시간 채팅** | 친구들끼리 수다 떨면서 플레이하는 재미 | LOW | 텍스트 기반, WebSocket 채널 공유 가능 |
| **사운드 효과 (효과음)** | 카드 돌리기, 베팅, 승리 시 피드백 강화 | LOW | 브라우저 Web Audio API, 나중에 추가 가능 |
| **족보 힌트/가이드** | 초보자가 족보를 몰라도 자기 패 강도를 알 수 있음 | LOW | 자기 패 옆에 족보 이름 + 순위 표시 |
| **게임 로그/히스토리** | 이전 판 결과 확인 (세션 내), "아까 그 판" 회상 가능 | LOW | 세션 메모리에 저장, 간단한 리스트 UI |
| **이모지/빠른 반응** | 채팅 대신 간단한 감정 표현 (웃음, 분노, 놀람 등) | LOW | 미리 정의된 이모지 버튼 5~6개 |
| **모바일 반응형 UI** | 친구들이 PC 없이 폰으로도 참여 가능 | MEDIUM | 터치 기반 인터랙션, 작은 화면 레이아웃 최적화 |
| **방장 게임 설정** | 시작 금액, 타이머 길이 등 방장이 커스터마이징 | LOW | 방 생성 시 설정 화면 |
| **패 공개 연출** | 결과 발표 시 순차적으로 카드 공개 (드라마틱한 효과) | MEDIUM | 타이밍 제어 + 순서별 애니메이션 |

### Anti-Features (의도적으로 만들지 않을 기능)

매력적으로 보이지만 이 프로젝트에서는 문제를 일으킨다.

| Feature | 왜 요청될 수 있는가 | 왜 문제인가 | 대안 |
|---------|-------------------|------------|------|
| **회원가입/계정 시스템** | "전적 저장", "프로필" 기대 | 핵심 가치가 "링크 하나로 즉시 플레이". 인증은 마찰 | 닉네임만 입력, 세션 기반 운영 |
| **AI/봇 플레이어** | "인원 부족할 때" | 친구들끼리 플레이가 목적. AI 개발은 복잡도 폭증 (섯다 AI 전략은 비자명) | 2인 최소 지원으로 충분 |
| **실제 현금 결제** | "진짜 돈 걸고 하고 싶다" | 법적 이슈 (도박법), 결제 인프라 복잡도, 프로젝트 범위 초과 | 가상 칩 + 오프라인 정산 |
| **관전자 모드** | "구경만 하고 싶다" | 카드 정보 노출 위험, 추가 상태 관리 복잡도, v1에서 불필요 | 빈 자리에 앉아서 다음 판 대기 |
| **글로벌 랭킹/리더보드** | "경쟁 재미" | 영구 저장소 필요, 계정 시스템 의존, 소규모 친구 그룹에 불필요 | 세션 내 칩 현황이 충분한 경쟁 지표 |
| **네이티브 모바일 앱** | "앱스토어에서 다운로드" | 개발/배포 비용 과다, 웹 앱 모바일 브라우저로 충분 | PWA 또는 반응형 웹으로 모바일 지원 |
| **실시간 화상/음성 채팅** | "얼굴 보면서 하고 싶다" | WebRTC 구현 복잡도 극심, 서버 리소스 | 카카오톡/디스코드 통화 병행 안내 |
| **다국어 지원** | "외국 친구도 같이" | 타겟이 한국인 친구 그룹, 섯다 자체가 한국 게임 | 한국어 단일 언어 |
| **복잡한 애니메이션/3D** | "고퀄리티 비주얼" | 와이어프레임 우선 전략과 충돌, 개발 시간 대비 가치 낮음 | 깔끔한 2D 와이어프레임 -> 추후 이미지 교체 |

## Feature Dependencies

```
[WebSocket 실시간 인프라]
    ├──requires──> [방 생성/참여 시스템]
    │                  └──requires──> [닉네임 입장]
    ├──requires──> [게임 상태 동기화]
    │                  ├──requires──> [카드 셔플/배분]
    │                  ├──requires──> [턴 관리]
    │                  └──requires──> [베팅 시스템]
    └──requires──> [연결 끊김 처리]

[카드 덱 (20장)]
    └──requires──> [족보 판정 엔진]
                       ├──requires──> [구사/멍텅구리구사 재경기 트리거]
                       ├──requires──> [땡값 정산]
                       └──requires──> [게임 결과 화면]

[오리지날 모드]
    └──enhances──> [세장섯다] (3장 중 2장 선택 추가)
    └──enhances──> [한장공유] (공유패 선택 UI 추가)
    └──enhances──> [후회의섯다] (20장 공개 + 카드 선택 UI)
    └──enhances──> [인디언섯다] (카드 가시성 반전 로직)

[턴 관리] ──enhances──> [타이머]

[게임 로그] ──requires──> [게임 결과 화면]

[채팅] ──independent──> (WebSocket 인프라만 공유, 게임 로직과 독립)
```

### 의존성 노트

- **WebSocket 인프라가 모든 것의 기반:** 방 관리, 게임 동기화, 채팅 모두 WebSocket 위에 구축. 이것이 가장 먼저 안정화되어야 함.
- **족보 판정 엔진은 게임의 핵심:** 모든 게임 모드가 최종적으로 족보 비교로 승패를 결정. 이 엔진이 정확해야 나머지가 의미 있음.
- **오리지날 모드가 다른 모드의 기반:** 세장섯다, 한장공유, 후회의섯다, 인디언섯다 모두 오리지날의 베팅/족보 시스템을 공유하면서 카드 배분 방식만 다름. 오리지날 먼저 완성 후 확장이 자연스러움.
- **채팅은 독립적:** 게임 로직과 무관하게 구현 가능. 어느 시점에든 추가할 수 있음.
- **땡값과 94재경기는 족보 판정에 의존:** 판정 결과를 기반으로 추가 로직이 동작하므로, 족보 판정 이후에 구현.

## MVP Definition

### Launch With (v1)

최소한으로 "친구들끼리 섯다를 온라인으로 플레이"를 검증할 수 있는 범위.

- [x] **WebSocket 실시간 인프라** -- 모든 기능의 기반
- [x] **방 생성 + 링크 초대 + 닉네임 입장** -- 핵심 가치 "링크 하나로 플레이"
- [x] **카드 덱 (20장) + 셔플/배분** -- 게임 시작의 전제
- [x] **족보 판정 엔진 (전체)** -- 섯다의 핵심, 부분 구현은 의미 없음
- [x] **오리지날 모드** -- 가장 기본, 다른 모드의 기반
- [x] **베팅 시스템 (콜/레이즈/다이)** -- 게임 인터랙션의 핵심
- [x] **칩 추적** -- 베팅에 필수
- [x] **원형 테이블 UI (와이어프레임)** -- 최소한의 게임 화면
- [x] **턴 관리 + 타이머** -- 무한 대기 방지
- [x] **게임 결과 화면** -- 판 종료 시 승자/패 공개
- [x] **연결 끊김 처리** -- 실사용에서 반드시 필요
- [x] **밤일낮장 선 결정** -- 하우스룰의 기본
- [x] **기리/퉁** -- 패 돌리기의 핵심 메커니즘

### Add After Validation (v1.x)

오리지날 모드가 안정적으로 동작한 후 추가.

- [ ] **세장섯다 모드** -- 오리지날 기반 + 3장째 배분 로직 추가
- [ ] **한장공유 모드** -- 공유패 선택 UI 추가
- [ ] **인디언섯다 모드** -- 카드 가시성 반전 (나만 못 봄 / 나만 봄) 로직
- [ ] **후회의섯다 모드** -- 20장 공개 + 순서대로 선택 UI (가장 복잡)
- [ ] **땡값 자동 정산** -- 오리지날 전용, 판정 후 추가 지불
- [ ] **94재경기 (구사/멍텅구리구사)** -- 재경기 트리거 + 재참여 UI
- [ ] **게임 내 채팅** -- 독립 기능, 언제든 추가 가능
- [ ] **족보 힌트** -- 자기 패 옆에 족보 이름 표시

### Future Consideration (v2+)

핵심 게임플레이 확인 후 고려.

- [ ] **사운드 효과** -- 몰입감 향상, 에셋 준비 필요
- [ ] **이모지/빠른 반응** -- 소셜 인터랙션 강화
- [ ] **패 공개 연출** -- 결과 발표 드라마틱 효과
- [ ] **게임 로그/히스토리** -- 세션 내 이전 판 기록
- [ ] **화투 이미지 에셋 교체** -- 와이어프레임 -> 실제 카드 이미지
- [ ] **방장 고급 설정** -- 타이머 길이, 시작금액 외 추가 옵션
- [ ] **모바일 최적화** -- 터치 인터랙션 전용 UI 개선

## Competitor Feature Analysis

| Feature | 한게임 섯다 | 넷마블 고스톱 | 포커 사이트 (PokerStars 등) | 우리 접근 |

| 계정 시스템 | 필수 가입 | 필수 가입 | 필수 가입 | **없음** -- 닉네임만, 마찰 제거가 차별점 |
| 매칭 시스템 | 랜덤 매칭 | 랜덤 매칭 | 로비 기반 | **링크 초대만** -- 친구 전용 |
| 실제 화폐/아이템 | 유료 아이템 | 유료 아이템 | 실제 현금 | **가상 칩만** -- 오프라인 정산 |
| 게임 모드 | 1~2종 | 1종 | 다양 (Hold'em, Omaha 등) | **5종** -- 하우스룰 전체, 이것이 핵심 차별점 |
| 채팅 | 정형 문구 | 정형 문구 | 텍스트 채팅 | 텍스트 자유 채팅 |
| 모바일 | 앱 전용 | 앱 전용 | 웹+앱 | **웹 전용** -- 설치 불필요 |
| 하우스룰 | 없음 | 없음 | 홈 게임 기능 있음 | **하우스룰이 기본** -- 땡값, 94재경기, 밤일낮장 등 |
| 스케일 | 대규모 | 대규모 | 대규모 | **소규모** -- 친구 그룹 전용 |

### 핵심 차별 포인트

1. **제로 마찰 진입**: 가입 없이 링크만으로 즉시 플레이
2. **완전한 하우스룰 지원**: 상용 게임에 없는 5가지 모드 + 땡값 + 94재경기 + 밤일낮장
3. **친구 전용 설계**: 매칭 시스템 대신 링크 초대, 가상 칩으로 오프라인 정산
